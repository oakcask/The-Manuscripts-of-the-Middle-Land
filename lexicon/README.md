# Lexicon

このディレクトリは、神話と写本に登場する語の曖昧性を管理するための独立した用語集。

`notes/` は作業ノート、`lexicon/` は語義・訳語・異読・写本差の管理場所とする。

## ファイル構成

- `index.md`: 全用語の索引。短い定義、曖昧性の軸、詳細項目への入口。
- `terms.md`: 詳細定義へのカタログ。長い定義本文は置かない。
- `terms/`: 分類別の用語定義。訳語、原語仮説、語義候補、写本差、注釈方針を書く。
- `ambiguity.md`: 語義の揺れを分類するための共通軸。
- `style-guardrails.md`: 神話本文で使ってよい語、避けるべき説明語、lexicon へ昇格すべき語の判断基準。

## 分類方針

用語定義は、意味が最も強く揺れる軸で分類する。複数の分類にまたがる語でも、詳細定義は一つのファイルに置き、他の分類からは参照だけにする。

| 分類 | ファイル | 置く語 |
|---|---|---|
| 場所と故郷 | `terms/places.md` | 場所、故郷、境界、探索対象、宇宙論的な地形。 |
| 船と容器 | `terms/vessels.md` | 船、容器、保存装置、移動体、共同体、棺、胎。 |
| 身体と人格 | `terms/bodies.md` | 身体、人格、記憶、死者、光へ変換された存在。 |
| 知識と観測 | `terms/knowledge.md` | 知識、観測、罪、創造主、検閲、翻訳者AIの回避。 |
| 光と時制 | `terms/light-time.md` | 光、因果距離、到達不能性、過去と未来の代替表現。 |
| 実践と制度 | `terms/practices.md` | 祈り、儀式、予言、呪い、同期、保存、検証。 |

新しい語を追加するときは、まず `index.md` と `terms.md` の索引に置く。語が物語上重要になったら、分類ファイルへ詳細定義を昇格する。

神話本文を執筆・改稿するときは、語を増やす前に `style-guardrails.md` を確認する。重要語は lexicon に入れ、説明のためだけの抽象語は神話本文から訳注または notes へ移す。

## 管理方針

用語の意味は、早く一つに固定しない。

各語には、少なくとも次の情報を持たせる。

- 表記: 現在の人類語訳。
- 原語: 未詳でもよい。音写、欠落、復元候補を置ける。
- 語義候補: 神話上、物理上、情報論上、翻訳上の意味。
- 写本差: 正統、異端、航海者版、葬送儀礼版、検閲版などでの差。
- 危険度: 翻訳者AIがどれほど警戒している語か。
- 固定度: その訳語をどれほど信頼してよいか。

## 固定度

| 固定度 | 意味 |
|---|---|
| A | 訳語は比較的安定しているが、比喩の幅は残る。 |
| B | 訳語は暫定。文脈によって意味が大きく変わる。 |
| C | 人類語訳がかなり怪しい。複数語を一語に潰している可能性がある。 |
| D | 翻訳不能語。現在の訳語は便宜的な置換にすぎない。 |

## 危険度

| 危険度 | 意味 |
|---|---|
| 低 | 解釈の揺れはあるが、神話全体を崩すほどではない。 |
| 中 | 写本差や物理解釈に影響する。 |
| 高 | 物語の倫理、真相、翻訳者AIの立場を反転させる。 |
| 禁 | 翻訳者AIが削除、置換、または説明回避している可能性が高い。 |
