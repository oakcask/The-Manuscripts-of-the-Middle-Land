# Story Planning Notes

このディレクトリは、短編ごとのアウトライン、プロット、場面設計、視点メモ、結末候補、本文に入る前の足場を置く場所。

実際の短編本文ドラフトはトップレベルの `stories/` に置く。ここには本文そのものではなく、本文を書くための設計資料を置く。

## 使い分け

- `index.md`: 複数短編をまたぐ構成、進行中の短編リスト、短編集全体の配置案。
- `001-title.md`: 個別短編のプロット、場面順、人物視点、結末候補。
- `fragments.md`: まだ個別短編に割り当てていない断片的な場面案や台詞案。

`notes/04-story-seeds.md` は短編集全体の候補リストとして残す。ある短編を具体的に進める段階になったら、対応する計画ファイルをこのディレクトリに作る。

## 短編タイトル

短編タイトルは、対応する神話章の名前をそのまま使うことを初期値にしない。

優先する候補は次のどちらか。

- 短編内で具体化される事件、喪失、危険、証拠、責任、観測結果を反映した題。
- 元になる神話本文から採った、短編の中心事件を強く示唆する一文または句。

神話章名を使うのは、それ自体が短編の具体的内容や視点上の圧力を十分に示している場合に限る。

個別短編の計画ノートには、仮題だけでなく、なぜその題が神話章名の単なる流用ではないかを短く残す。

## 複数視点プロット

短編プロットを作るときは、原則として一つの語り手に固定しない。

個別短編の計画ノートでは、少なくとも複数の視点候補を作る。各視点は、単なる語り手の変更ではなく、事件の意味、中心になる損失、恐怖、証拠、責任の所在が変わるように設計する。

各視点には次を含める。

- 語り手ペルソナ
- 対応する写本立場
- 事件の見え方
- 場面順
- 結末
- 翻訳者注で反転または不安定化させる点

共有される事件の核は冒頭にまとめる。ただし、どの視点が真相かは確定しない。複数視点の差異を、後で複数の神話写本を構成する材料として扱う。

## 短編本文の読みやすさ

トップレベルの `stories/` に置く短編本文では、英語版と日本語版のどちらも短編としての読みやすさを優先する。

神話本文のような直訳調、儀礼的な硬さ、用語の圧力は、短編内で必要な場所に限って使う。人物の行為、観測、会話、場面転換では、自然な英語・自然な日本語に寄せてよい。

特に日本語版では、英語の語順や比喩を硬く写して読みにくくしない。未登録の普通の表現は、`lexicon/style-guardrails.md` の日本語訳方針に従い、意味と場面が伝わる言い回しを選ぶ。
